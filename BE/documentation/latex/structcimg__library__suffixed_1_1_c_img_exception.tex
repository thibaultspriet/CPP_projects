\doxysubsection{Detailed Description}
Instances of {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_exception}{CImg\+Exception}}} are thrown when errors are encountered in a \textbackslash{}\+CImg function call. 

\begin{DoxyParagraph}{Overview}

\end{DoxyParagraph}
\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_exception}{CImg\+Exception}} is the base class of all exceptions thrown by \textbackslash{}\+CImg (except {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_abort_exception}{CImg\+Abort\+Exception}}}}). \mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_exception}{CImg\+Exception}} is never thrown itself. Derived classes that specify the type of errord are thrown instead. These classes can be\+:


\begin{DoxyItemize}
\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_abort_exception}{CImg\+Abort\+Exception}}\+:}} Thrown when a computationally-\/intensive function is aborted by an external signal. This is the only {\ttfamily non-\/derived} exception class.
\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_argument_exception}{CImg\+Argument\+Exception}}\+:}} Thrown when one argument of a called \textbackslash{}\+CImg function is invalid. This is probably one of the most thrown exception by \textbackslash{}\+CImg. For instance, the following example throws a {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_argument_exception}{CImg\+Argument\+Exception}}\+:} 
\begin{DoxyCode}{0}
\DoxyCodeLine{CImg<float> img(100,100,1,3); \textcolor{comment}{// Define a 100x100 color image with float-\/valued pixels}}
\DoxyCodeLine{img.mirror(\textcolor{charliteral}{'e'});              \textcolor{comment}{// Try to mirror image along the (non-\/existing) 'e'-\/axis}}

\end{DoxyCode}

\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_display_exception}{CImg\+Display\+Exception}}\+:}} Thrown when something went wrong during the display of images in CImg\+Display instances.
\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_instance_exception}{CImg\+Instance\+Exception}}\+:}} Thrown when an instance associated to a called \textbackslash{}\+CImg method does not fit the function requirements. For instance, the following example throws a {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_instance_exception}{CImg\+Instance\+Exception}}\+:} 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{const} CImg<float> img;           \textcolor{comment}{// Define an empty image}}
\DoxyCodeLine{\textcolor{keyword}{const} \textcolor{keywordtype}{float} value = img.at(0);   \textcolor{comment}{// Try to read first pixel value (does not exist)}}

\end{DoxyCode}

\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_i_o_exception}{CImg\+IOException}}\+:}} Thrown when an error occurred when trying to load or save image files. This happens when trying to read files that do not exist or with invalid formats. For instance, the following example throws a {\ttfamily \mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_i_o_exception}{CImg\+IOException}}\+:} 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{const} CImg<float> img(\textcolor{stringliteral}{"{}missing\_file.jpg"{}});  \textcolor{comment}{// Try to load a file that does not exist}}

\end{DoxyCode}

\item {\bfseries{\mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_warning_exception}{CImg\+Warning\+Exception}}\+:}} Thrown only if configuration macro {\ttfamily cimg\+\_\+strict\+\_\+warnings} is set, and when a \textbackslash{}\+CImg function has to display a warning message (see \mbox{\hyperlink{namespacecimg__library__suffixed_1_1cimg_acfd3624d72ed7f79c82ec45646c685e7}{cimg\+::warn()}}).
\end{DoxyItemize}

It is not recommended to throw \mbox{\hyperlink{structcimg__library__suffixed_1_1_c_img_exception}{CImg\+Exception}} instances by yourself, since they are expected to be thrown only by \textbackslash{}\+CImg. When an error occurs in a library function call, \textbackslash{}\+CImg may display error messages on the screen or on the standard output, depending on the current \textbackslash{}\+CImg exception mode. The \textbackslash{}\+CImg exception mode can be get and set by functions cimg\+::exception\+\_\+mode() and \mbox{\hyperlink{namespacecimg__library__suffixed_1_1cimg_aeafab144366eddd11360f32f468fa4af}{cimg\+::exception\+\_\+mode(unsigned int)}}.

\begin{DoxyParagraph}{Exceptions handling}

\end{DoxyParagraph}
